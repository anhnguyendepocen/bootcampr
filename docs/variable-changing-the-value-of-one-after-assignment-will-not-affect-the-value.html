  • 49 Functions
  • 50 ———–
  • 51 If you have certain tasks that you do a lot in R, the best way to save time is
  • 52 to automate these tasks by writing functions. This may sound daunting, but it
  • 53 really isn’t- functions have only three key parts: argument, body, and return
  • 54 The way you create a function in R is very similar to how you would create a
  • 55 variable. For this function, we are just multiplying a number by two
  • 56 You can also make functions that take multiple inputs, or arguments (but you
  • 57 don’t have to name them “argument” like I did in the first example; you do
  • 58 have to use the word “function” when creating a function though, and you
  • 59 should also use “return” at the end of the function)
  • 60 But it is important to note that named arguments are only accessible within
  • 61 the function where they appear
  • 62 …the same goes for variables that are initialized within functions
  • 63 This is due to a phenomenon called lexical scoping: variables will only be
  • 64 available or changeable in the the context in which they are created (and all
  • 65 contexts that are built from that context). This means that a variable that is
  • 66 initialized outside of a function (in the “global” scope) will be available
  • 67 in both the global and function scope; remember that we initialized z to -2
  • 68 …but if we change the value of a global variable in the function scope, this
  • 69 will not carry back over to the more general global scope
  • 70 One last thing about functions for today: you can save the return values of
  • 71 functions as variables- this will make it so that you don’t have to
  • 72 recalculate functions each time you want to use the same output
  • 73 There’s a lot more that you can do with functions, and we’ll come back to
  • 74 functions throughout the course as we get deeper into R’s capabilities!
